Complete Rewrite

Most of the original debug modes codes are very messy in design. This rework completely shifts logic from inside the gecko pass to individual renderDebug functions of each object. This way of working makes it so it is very generalisable. Hitbox renderring was written for fighters but automatically works for hazards on stages as well as subspace enemies and bosses.

Clean Graphics

Through using the games graphics engine, graphics are drawn directly to the screen instead of having to utilise gfx objects, removing limitations on number of hitboxes/effects on screen at once, as well as allow much more accurate display to be possible, e.g. interpolation between frames/between two ends of a hurtbox.

Collision Display:

Allows users to view how the game interprets hitboxes and hurtboxes to better understand whether something should hit and how interactions play out
Also allows completely disabling the fighters render so that its easier to see the collisions.

completion
Status Module Display:

Prints text to the screen detailing the exact action each fighter is in, as well as details on their subaction and what frame of animation they are on.

Stage Collisions Display:

Allows you to disable the stage render entirely and instead draw the exact stage collisions the game uses.
This also allows viewing blastzones and camera boundaries.
